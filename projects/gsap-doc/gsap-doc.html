Intro to GSAP

Go crazy with animation!

GSAP is a JavaScript library created by GreenSock. It's a simple option for creating complex animations on your website. This library is easy enough for JS newbies and robust enough for experienced front-end devs.

Every animation on this page was created with GSAP!

This guide is a "quick start" to get you animating. If you're a seasoned web dev, you might be comfortable jumping straight into the official GreenSock docs. If you're inexperienced with JavaScript – perhaps you're learning web design right now – this is the guide for you.

Fun Facts About GSAP:
+
  • GSAP is usually used to animate DOM elements, but it can animate properties of almost anything, from SVGs, to canvas elements, to webgl, to JS objects.
  • GSAP lets you focus on creating animations instead of fumbling around with complexities like measurement units like pixels and rems and view widths.
  • You can use GSAP with other libraries like React.
  • You can even use GSAP with Wordpress.
  • GSAP offers its core features for free. If you want even more animation and customizations, you can join Club GreenSock. This is a membership program that offers tiers of bonus functionality and features. You can even get a special commercial license for your business to implement in its own projects.

The Basics

First: a script tag

Since GSAP is a JavaScript library, you first need to bring it into your code. The easiest way is with a <script> tag at the end of your HTML <body>. This is a quick way to get started, plus it loads fast and is widely cached around the internet—many websites use it. The GreenSock homepage has the most current source link.

The animations on this page uses:
<script src="https://cdn.jsdelivr.net/npm/gsap@3.12.2/dist/gsap.min.js"></script>

Second: tweens and timelines

Tweens:

Single animations are done with "tweens". A tween is the basic tool for creating an animation with GSAP. It's called a tween because it controls what happens to something in between states, or in between the animation's starting and stopping points. In your JavaScript, a basic tween looks like this: gsap.from(".gsap-logo", {x: -1000, duration: 5}). Tweens have three components: the method, the target, and the variables.

  • The Method: The type of action that you're animating. Tweens have four methods:
    • gsap.to(): Moves something from its current state "to" the values defined in the tween's variables. The current state is typically defined in CSS.
    • gsap.from(): Moves something "from" the values defined in the tween's variables to its current state. The current state is typically defined in CSS.
    • gsap.fromTo(): A combination of the previous two, so you get to define both the starting and ending values.
    • gsap.set(): Immediately sets the properties from the variables so there is no animation, as if the animation happened instantly. This may be useful if you want to reverse the tween.
  • The Target: The object that you want to animate, like a DOM object or image. In the basic tween example above, ".gsap-logo" is an element in the HTML with a class of .gsap-logo.

    In your tweens, don't forget to reference classes with ".[class name]" and ids with "#[id name]" (you must include the "#" for ids).

  • The Variables: The properties of the element that you want to change or animate. These can correlate with CSS properties. GSAP also offers special properties like duration or onComplete. In the basic tween example above, the target element will move from its starting position of -1000px (x-axis) to its final position defined in the CSS. It will also take five seconds to complete.

Timelines:

A timeline is a sequence of tweens. It lets you piece together individual tweens in a specific order and easily control their timings. This is a simple way to create a cohesive series of animations. By default, each tween in a timeline starts after its predecessor ends, so they play one after another. You could accomplish the same cohesive animation with a series of individual tweens. However, without a timeline, you might need to add complicated duration and delay variables for each tween to produce your desired result. This could become confusing as you add more and more tweens. Timelines simplify this entire process.

Another benefit of timelines is that you can control the entire timeline as a singular entity. Each tween within the timeline is an individual animation, but the entire timeline can also be controlled. For example, you could reverse the timeline so that it plays backwards, or in reverse.

Tween Animation Example

This section explains a tween animation on this page: the GSAP logo sliding into the screen.

This animation was created with a single tween. The tween fires immediately when the window loads, using a standard JS event listener. The CSS determines where the logo should be positioned after the animation. This allowed the gsap.from() method to be used. The "from" method determines from where the tween begins moving and the CSS determines its final position.

Here is the exact tween used: 

window.addEventListener('load', function() {
	gsap.from(".gsap-logo", {x: -1000, duration: 5, ease: "elastic.out(1, 0.5)"});
})

Key details about the tween:

  • gsap.from() is the method.
  • The GSAP logo image has the .gsap-logo class applied in the HTML. This allows the tween to target the image with .gsap-logo.
  • The x property starts the tween at -1000 pixels from it's final position. The image moves from left to right along the x-axis to it's final position. The duration is 5 seconds, so it takes 5 seconds for the animation to complete. The ease variable makes the logo look like it's jostling or jerking into position. GSAP uses this ease variable to make it seem like the animation has actual momentum – like there is weight behind it. ease is one of the easiest ways to enhance your animations, giving the animation and your site more character.

Timeline Animation Example

This section explains a timeline animation on this page: the navigation options jumping into position one-by-one.

This animation fires immediately when the window loads, using a standard JS event listener. The timeline fires along with the tween animation of the GreenSock logo, explaind above. A timeline was used to sequentially animate each individual item in the header, including "Sean Patterson" text and the navigation items. Each element was given an id. These ids are used as the targets in the GSAP statement.

Here is the complete codeblock. You can see the event listener added to the window, the individual logo tween, then the timeline. 

window.addEventListener('load', function() {
	gsap.from(".gsap-logo", {x: -1000, duration: 5, ease: "elastic.out(1, 0.5)"});
	
	let tl = gsap.timeline({defaults:{opacity: 0, ease: "elastic.out(1, 0.75)", duration: 1}})
	tl.from("#one", {y: -50})
	.from("#two", {y: 150}, "-=.5")
	.from("#three", {y: -50}, "-=.5")
	.from("#four", {y: 150}, "-=.5")
	.from("#five", {y: -50}, "-=.5")
})

Key details about the timeline:

  • GSAP timelines are declared as a variable called tl.
  • In the variable declaration, three default properties were set: opacity, ease, and duration. These defaults are applied to every individual tween in the timeline.
  • After declaring the timeline variable with its defaults, the next line starts the animation with the first tween. This tween must begin with the tl object. The first tween does not end with a semicolon, therefore each following tween can continue with just the .from() method. This is called tween chaining.
  • Each of the tweens use the .from() method. Just like the logo tween above, the CSS defines the final position of the elements, and GSAP controls from where the animation starts.
  • By default, each tween in the timeline fires after the previous one concludes. You can offset this so that the timeline tweens start before previous ones conclude. The tweens in this timeline are offset by half a second. After the tween variables are defined, the tweens have the offset defined with , "-=.5"

The logo and header elements are all animating at the same time. Therefore, the individual tween that controls the logo animation could have also been included in the timeline to keep everything together. The logo tween was separated out for the sake of an example.

Plugins

GSAP offers many free plugins to add more animation options. GreenSock created these features as plugins to keep the core as small as possible, making it as fast as possible. The plugins are for unique situations that the core functionality may not cover. This is why they're offered as plugins; if you want to add certain functionality, you only need to use a single plugin, keeping your code slim and efficient.

Explore GSAP v3 plugins

Some plugins are already built into the core functionality of GSAP. For example, the AttrPlugin, which lets you tween an unlimited number of attributes simultaneously. Another commonly used plugin is CSSPlugin, which lets you animate almost any CSS-related property of DOM elements, even if their values are not numeric.

Commonly Used Features

The GSAP library is extensive. It allows professional developers to create advanced and complex animations. However, you do not need to be a tenured dev to create something awesome with GSAP. The following sections describe features of GSAP that are commonly used and easy enough for novices to use.

Ease

Ease gives personality to your animation by controlling the speed or timing of the movement. It was used for the animations on this page, like the navigation options bouncing into place. The GreenSock logo also "eases" into place like a ball slowly bouncing less and less as it loses energy. You can add this behavior to your animation with the ease variable in your tween. GSAP contains default types of ease that you can use as values, like elastic or bounce. You can also create custom ease behavior with CustomEase.create() as the value.

You can play around with different types of ease or make a custom ease on the GreenSock website.

Reverse

You can reverse an animation, both individual tweens or full timelines. This saves you time as you only need to create the animation one time, then you can reverse it without needing to replicate the animation in the opposite direction. For example, the header elements on this site bounce into place one at a time. Each of the five elements are assigned an id of one, two, three, four, and five. The timeline plays through the animation, starting with one and ending with five. If this timeline were reversed, the animation would play it back the exact opposite way, so starting with the end of animation #five back to the beginning, then #four, and so on. Everything about the animation is played backward, even the ease.

Callback Functions

Callback functions let you fire another type of behavior when a certain event occurs for an animation. For example, you could use the onComplete callback to run a function when the animation finishes. You can use these callbacks to run any function that you write.

You include these callback functions inside of a tween. For example, the following tween would take five seconds to rotate 360 degrees, then log "Rotation finished!" to the console after it's finished. 

gsap.to(".gsap-logo", {duration: 5, rotation 360, onComplete: function() {
	console.log("Rotation finished!");
}});

The following is a list of easy to use callbacks:

onComplete
When a single animation finishes.
onUpdate
Any time the animation updates or changes. Using the rotation example above, if you used onUpdate to log the angle of rotation, each degree of change would be logged.
onStart
When a single animation starts.
onReverseComplete
When an animation or timeline is reversed, then completes.
onRepeat
When an animation starts again as a repeat.

CSS Properties With Hyphens

You can animate almost every CSS property with GSAP. However, the CSS property names are not exactly the same when referenced in GSAP. For properties that are multiple words and use a hyphen between the words, like background-color, you would write it with camelCase instead. So, background-color in CSS becomes backgroundColor in GSAP.
